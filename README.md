#Radar Management – B2B Lead Enrichment Microservice

Микросервис для автоматического поиска, сбора и обогащения данных о компаниях через API Checko.
Позволяет по коду ОКВЭД найти организации, получить их контакты, финансовые показатели и сохранить в PostgreSQL.
Предназначен для построения базы потенциальных клиентов («лидов») и последующего скоринга (оценки приоритета) с помощью AI.

# Основные возможности

* Поиск компаний по ОКВЭД (с фильтром только действующих).

* Сохранение базовых сведений (ИНН, название, статус, ОКВЭД, дата регистрации).

* Обогащение контактами: телефоны, email, веб-сайт, регион.

* Обогащение финансовыми данными: выручка за 2023–2025 годы.

* Единый эндпоинт /enrich для полного обогащения (контакты + финансы) одной компании.

* Готовые REST API для выборочного обновления и просмотра.

* Строгая типизация API через Pydantic Response Models с автоматической валидацией ответов и документацией Swagger.

* HTTP-запросы (httpx) для высокой производительности.

* **AI‑оценка (скоринг) компаний** через OpenRouter – автоматический расчёт приоритетности лида на основе финансовых показателей,

региона и других факторов. Модель возвращает уровень риска, приоритет лида и краткое текстовое обоснование.


# Стек технологий
* Python 3.11+

* FastAPI – веб‑фреймворк, автодокументация (Swagger/ReDoc)

* SQLAlchemy 2.x 

* PostgreSQL – основная база данных

* httpx – HTTP‑клиент

* Pydantic – схемы запросов и ответов (Request/Response Models), валидация данных и генерация OpenAPI/Swagger документации.

* Alembic – миграции схемы БД

* **OpenRouter** – агрегатор LLM (Gemma др.) для AI‑аналитики

* **Автоматический расчёт динамики роста** – при запросе списка компаний API вычисляет процентный прирост выручки и прибыли
относительно предыдущего года по формуле:  
`(текущее_значение - предыдущее_значение) / |предыдущее_значение| * 100`  
с корректной обработкой нулевых и отрицательных значений.

## API Validation

Проект использует Pydantic Response Models для всех основных REST API.

Это обеспечивает:

- автоматическую валидацию ответов FastAPI;
- генерацию подробной OpenAPI/Swagger документации;
- строгую типизацию данных между сервисным слоем и API;
- защиту от возврата некорректной структуры ответа.


##  Динамика роста (growth_calc)

В эндпоинте `GET /companies` для каждой компании вычисляются два дополнительных поля:
- `revenue_growth` – процентный рост выручки (2025 vs 2024)
- `profit_growth` – процентный рост прибыли (2025 vs 2024)

Функция расчёта `growth_calc(current, previous)`:
- Если `previous` или `current` равны `None` → возвращает `None`.
- Если `previous == 0` и `current == 0`  возвращает `0`.
- Если `previous == 0` и `current != 0`  возвращает `100` (условный бесконечный рост).
- Иначе: `round((current - previous) / abs(previous) * 100, 1)`.

Это позволяет корректно обрабатывать ситуации, когда в предыдущем году показатели были нулевыми или отрицательными.

##  Интеграция с AI (OpenRouter)

Проект использует [OpenRouter](https://openrouter.ai) как единый шлюз к бесплатным и платным LLM.  
Это позволяет:
- Получать аналитические выводы по финансовой отчётности компании.
- Автоматически рассчитывать **скоринг приоритета** (от 0 до 100) на основе выручки, прибыли, динамики роста и региона.
- Генерировать краткое описание компании для менеджеров по продажам.
